String类、StringBuilder类、StringBuffer类

  • 都可以表示一个字符串
  • 区别：
  • 1.运行速度： StringBuilder > StringBuffer > String
  • 原因： String是字符串常量,String对象一旦被创建,不可更改。StringBuilder和StringBuffer都是字符串变量
  • 2.线程安全： StringBuilder线程不安全,但是效率高,StringBuffer线程安全,但是效率低
  • 总结：
  • 1.String类适用于少量字符串操作
  • 2.StringBuilder类适用于单线程下大量字符串操作
  • 3.StringBuffer类适用于多线程下大量字符串操作

多线程的数据访问冲突

一个线程访问到修改了一般的数据，有新数据，也有旧数组，这种数据成为脏数据

  • 解决方法—–线程同步 synchronized

    • 同步：多个线程，步调一直的执行

    • 让多个线程，争夺同一个对象的“同步锁”

      谁抢到谁执行，抢不到则等待

    • 同步锁

      • 任何对象，都有唯一的一个同步锁

    • synchronized关键字

      • 遇到同步关键字synchronized，要先抢到锁才能执行，抢不到则等待

    • synchronized用法：

      1. synchronized(对象){

}//争夺指定对象的锁

      2. ```java
        A1 a1 = new A1();
        A1 a2 = new A1();
        a1.f();
        a2.f();
        static synchronized void f(){

        }//抢类（A1）的锁

  • 参数1：表示核心线程数
  • 参数2：表示最大线程数（包括核心线程数）
  • 参数3：表示最大线程数的存活时间（不包括核心线程数）
  • 参数4：表示存活时间的单位（枚举类型）
  • 参数5：表示线程堵塞（等待）队列，允许处于等待状态的线程数
  • 参数6：表示创建线程的工厂模式，默认是正常优先级，非守护线程
