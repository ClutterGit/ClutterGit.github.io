在win10家庭版环境下更改C盘中的用户名

发表于 2022-04-11 | 分类于 Windows

笔记背景

因年少无知，当初把电脑用户名设置成了中文，因为一直不碍事就没研究，最近因为装nvm无法install下载管理nodejs版本，百度了一番，发现是因为C:\user下的用户名为中文，在cmd窗口下乱码了，无法找到这个文件路径，从而无法下载安装nodejs。

笔记目标

将window10家庭版系统中C:\user\【我的名字】改为C:\user\【MyUserName】目录下的用户名修改为英文,我的名字原中文名称，【MyUserName】为修改后的英文名称，你可以改为自己想要的英文名称。亲测有效！

第一步

打开控制面板进入用户账户界面点击更改账户名称，将名称改为【MyUserName】。

第二步

win+R输入regedit，进入注册表编辑器，再打开目录‘计算机\HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows NT\CurrentVersion\ProfileList’，从ProfileList目录中挨个检查ProfileImagePath的值，若为C:\user\【我的名字】，打开ProfileImagePath修改它的值为C:\user\【MyUserName】。

第三步

重启计算机，计算机打开后会显示‘win10无法登陆到你的账户’让你注销或关闭，并且之前计算机中的文件都不显示了，此时你不用管他，直接win+E打开文件资源管理器，进入C:\user目录，这时你会发现多了一个Temp文件，并且【我的名字】还是中文，Temp文件是计算机为你自动创建的一个临时安全账户文件方便你进入你的计算机，这时你右键【我的名字】会发现文件可以重命名了，记住名称一定要改成之前在账户页面以及注册表编辑器中改的【MyUserName】。这样重启电脑就发现
你的文件都回来了，而且C:\user\【我的名字】也变成了C:\user【MyUserName】。

注意：

因为计算机之前在使用的时候很多程序都要配置环境变量，此时计算机中的某些程序会运行不了。所以要右键我的电脑—>属性—>高级系统设置—>环境变量，将其中用户变量与系统变量中的【我的名字】改为【MyUserName】。

数据结构

概论

  • 数据结构是相互之间存在一种或多种特定关系的数据元素的集合。

    • 数据：描述客观事物的符号，是计算机中可以操作的对象，是能被计算机识别，并输入给计算机处理的符号集合。

    • 数据元素：组成数据的、有一定意义的基本单位，在计算机中通常作为整体处理。

    • 数据项：一个数据元素可以由若干个数据项组成，是数据不可分割的最小单位。

    • 数据对象：性质相同的数据元素的集合，是数据的子集。

      理解==》数据是一个抽象的概念，然后数据对象包含于数据，数据对象<数据元素<数据项。

广义表

广义表是n(n>=0)个元素a1,a2,a3,…,an的有限序列，其中ai是原子项，或是一个广义表。
表示： LS=（a1,a2,a3,…,an)。
LS是广义表的名字，n为它的长度。若ai是广义表，则称它为LS的子表。
为了区别原子和广义表，广义表用大写字母表示， 原子使用小写字母。
例：D=(( ),(e),(a,(b,c,d))) D中含有三个广义表，在子表中还存在着子表
E=（a, E） 广义表自己引用自己是一个深度无限，大小为2的表

广义表的结构特点

①“原子”的深度为“0”; “空表”的深度为1；
②广义表的长度定义为最外层包含的元素个数；
③表头可以是原子或列表; 表尾必定是列表；
④广义表可以是一个递归的表; 递归表的深度是无穷值，长度是有限值；
⑤广义表的元素可以是子表，而子表的元素还可以是子表，广义表是一个多层次的结构；
⑥广义表可为其它表所共享。

广义表的存储结构

由于广义表中的数据元素可以具有不同的结构。因此，通常采用链式存储结构。
在定义的时候需要定义两种节点，一种表节点，一种原子节点。

树

常用树操作术语

  • 树的节点：包含一个数据元素及若干指向其它子树的分支。

  • 节点的度：节点拥有的子树个数。

  • 叶子节点（终端节点）：度为0的节点

  • 分支节点（非终端节点）：度不为0的节点

  • 树的度：树内各节点的度的最大值
  • 孩子、双亲：节点的子树的根称为该节点的孩子，相应该节点称为双亲。
  • 兄弟：同一个双亲的孩子之间互称兄弟
  • 子孙：某节点为跟的子树中的任意节点都成为该节点的子孙。
  • 祖先：从根到该节点所经分支上的所有节点。

  • 层次：从根开始定义起，根为第一层，根的孩子为第二层。
  • 深度（高度）：树中最大层次称为深度或高度。

  • 孩子兄弟表示法：一棵树的第一个孩子如果存在就是唯一的，他的右兄弟存在也是唯一的，因此可以用firstchild表示该节点的第一个孩子，用rightsib表示这个孩子的右兄弟。

图中的一些术语

  • 无向边：如果顶点到之间的边没有方向，称这条边为无向边，使用无序偶对表示。
  • 无向图：图中任意两个顶点之间的边都是无向边，称该图为无向图。
  • 有向边：如果顶点到之间的边有方向，称这条边为有向边，使用有序偶对表示。
  • 弧（弧头、弧尾）：A到D的有向边就是弧，A就是弧头，D就是弧尾，记为<A,D>
  • 稀疏图、稠密图：有很少条边或弧的图称为稀疏图，反之称为稠密图。
  • 简单图：若不存在到自身的边，且边不重复出现，称为简单图。
  • 完全图，任意两个顶点之间都存在边，称为完全图
  • 权：有些边或弧具有它相关的数字，称为权
  • 网：带权的图称为网。
  • 子图：假设有两个图和，如果且则称G’为G的子图。
  • 度：顶点的度，与顶点v相关边的数目，记为TD(V)。
  • 邻接：如果无向图的边则称顶点v和v‘互为邻接点，v和v’邻接；有向图中则称顶点v邻接到顶点v’。
  • 出度、入度：有向图中有向图中则以顶点v为头的弧的数目称为v的入度，记为ID(v)。以v为尾的弧的数目称为v的出度，记为OD(v)。TD(v)=ID(v)+OD(v)。
  • 路径：从顶点v到v’的路径是一个顶点序列，其中相邻序列为图中边。
  • 路径长度：路径上的边的数目
  • 简单路径：序列中顶点不重复出现的路径称为简单路径。
  • 回路、环：第一个顶点和最后一个顶点相同的路径称为回路或环。
  • 简单回路：除了第一个顶点和最后一个顶点之外，其余顶点不重复出现的回路称为简单回路。
  • 连通：图中从顶点v到顶点v‘有路径，则称v和v’是连通的。
  • 连通图：图中任意两个顶点都是连通的则图就是连通图。
  • 连通分量：无向图中的极大连通子图称为连通分量。
  • 生成树：生成树是一个极小的连通子图，含有图中全部n个顶点，但只有足以构成一棵树的n-1条边。
  • 生成森林：非联通图的每个图生成一个树后得到的就是生成森林。

最小生成树是构造连通网的最小代价生成树。本质就是把图中权大的边删除掉只留下n-1条边，或者说从权最小的边开始进行保留，不构成环的边留下生成的数据结构。

  • 普利姆算法：找一个顶点，查找连接该顶点权值最小的边。
  • 克鲁斯卡尔算法：直接查找权值最小的边。

最短路径：两个顶点之间经过的边上的权值之和最少的路径，第一个顶点称为源点，最后一个顶点称为终点。

  2. 活动最晚开始事件：结点右上角的值——权重
